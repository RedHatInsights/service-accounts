# Konflux E2E Setup Guide - Service Accounts

This guide walks you through the complete setup for running Playwright E2E tests in the Konflux pipeline for the service-accounts application.

## Overview

The Konflux E2E pipeline runs Playwright tests against the application on every pull request. It requires:

1. **ConfigMaps** - App Caddyfile and proxy routing (generated by Plumber)
2. **ExternalSecret** - Vault credentials for E2E test user
3. **Pipeline configuration** - Already in `.tekton/service-accounts-pull-request.yaml`

## Prerequisites

- Access to `konflux-release-data` repository (GitLab)
- Platform team support for Vault credentials
- Plumber installed locally

## Step 1: Install Plumber

Plumber generates the required ConfigMaps automatically. **Never edit ConfigMaps manually.**

```bash
git clone https://github.com/catastrophe-brandon/plumber.git
cd plumber

# Install (macOS: use pip3 if needed)
uv pip install -e .
# OR
pip install -e .
```

## Step 2: Generate ConfigMaps

Run Plumber from the plumber directory (or any directory - output goes to current dir):

```bash
plumber service-accounts \
  https://github.com/RedHatInsights/service-accounts.git \
  --app-configmap-name service-accounts-app-caddy-config \
  --proxy-configmap-name service-accounts-dev-proxy-caddyfile \
  --namespace rh-platform-experien-tenant \
  --frontend-yaml deploy/frontend.yml \
  --stage-env-url https://stage.foo.redhat.com
```

**Note:** If Plumber cannot find `frontend.yml` or `fec.config.js` (e.g., when running from plumber directory), the routes may be wrong. Run from the service-accounts repo directory or use absolute paths. Verify the generated ConfigMaps use `/apps/service-accounts` (not `/service-accounts`).

This generates:
- `service-accounts-app-caddy-config.yaml`
- `service-accounts-dev-proxy-caddyfile.yaml`

### Verify Generated ConfigMaps

Ask someone to review the ConfigMaps for:
- Correct route paths (`/apps/service-accounts*` for app assets)
- No overly broad routes
- Proxy ConfigMap should only contain app-specific routes (fallback to HCC_ENV_URL is handled by the pipeline)

## Step 3: Submit ConfigMaps to konflux-release-data

```bash
git clone git@gitlab.cee.redhat.com:releng/konflux-release-data.git
cd konflux-release-data

cp /path/to/service-accounts-app-caddy-config.yaml \
  tenants-config/cluster/stone-prd-rh01/tenants/rh-platform-experien-tenant/

cp /path/to/service-accounts-dev-proxy-caddyfile.yaml \
  tenants-config/cluster/stone-prd-rh01/tenants/rh-platform-experien-tenant/

# Edit kustomization.yaml to add:
# - service-accounts-app-caddy-config.yaml
# - service-accounts-dev-proxy-caddyfile.yaml

git checkout -b add-service-accounts-e2e-configmaps
git add tenants-config/cluster/stone-prd-rh01/tenants/rh-platform-experien-tenant/service-accounts-*.yaml
git add tenants-config/cluster/stone-prd-rh01/tenants/rh-platform-experien-tenant/kustomization.yaml
git commit -m "Add E2E ConfigMaps for service-accounts"
git push origin add-service-accounts-e2e-configmaps
```

Create a merge request.

## Step 4: Create Vault Credentials

Work with the platform team to create credentials at:
`creds/konflux/service-accounts`

Required properties:
- `username` - E2E test user
- `password` - E2E test password
- `e2e-hcc-env-url` - HCC environment URL (e.g., https://stage.foo.redhat.com)
- `e2e-stage-actual-hostname` - Stage hostname

Refer to the Platform Engineer Survival Guide for details.

## Step 5: Submit ExternalSecret to konflux-release-data

Use the template `service-accounts-credentials-secret.yaml` from this repository:

```bash
cd konflux-release-data

cp /path/to/service-accounts/service-accounts-credentials-secret.yaml \
  tenants-config/cluster/stone-prd-rh01/tenants/rh-platform-experien-tenant/

# Edit kustomization.yaml to add:
# - service-accounts-credentials-secret.yaml

git checkout -b add-service-accounts-e2e-credentials
git add tenants-config/cluster/stone-prd-rh01/tenants/rh-platform-experien-tenant/service-accounts-credentials-secret.yaml
git add tenants-config/cluster/stone-prd-rh01/tenants/rh-platform-experien-tenant/kustomization.yaml
git commit -m "Add ExternalSecret for service-accounts E2E credentials"
git push origin add-service-accounts-e2e-credentials
```

Create a merge request.

## Step 6: Pipeline taskRunSpecs (App Caddyfile)

The pipeline requires `taskRunSpecs` to mount the app Caddyfile. This is already configured in `.tekton/service-accounts-pull-request.yaml`:

- `service-accounts-app-caddy-config` ConfigMap mounted into the run-application sidecar
- `hostAliases` for stage.foo.redhat.com → localhost

## Troubleshooting

### Pipeline fails: ConfigMap not found
- Ensure ConfigMaps are merged to konflux-release-data and ArgoCD has synced
- Verify ConfigMap names match exactly: `service-accounts-app-caddy-config`, `service-accounts-dev-proxy-caddyfile`

### Pipeline fails: ExternalSecret not syncing
- Verify Vault credentials exist at `creds/konflux/service-accounts`
- Check ExternalSecret status in the cluster
- Ensure all 4 properties are present in Vault

### E2E tests fail: Page not loading
- Verify proxy routes are correct (`/apps/service-accounts*` → port 8000)
- Check that credentials (e2e-hcc-env-url) are set for proxy fallback

### Plumber generates wrong routes
- Use `--frontend-yaml deploy/frontend.yml` with correct path
- Run from service-accounts repo directory if paths are relative
- If routes are wrong, open an issue at https://github.com/catastrophe-brandon/plumber/issues
